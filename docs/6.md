# Инфологическая модель (ER-диаграмма)

## Сущности и атрибуты

- **Пользователь** (`User`)
  - `id` (уникальный идентификатор)
  - `username` (уникальное имя)
  - `password` (хэш пароля)
  - `status` (онлайн/офлайн)

- **Сообщение** (`Message`)
  - `id`
  - `senderId` (ссылка на отправителя)
  - `receiverId` (ссылка на получателя)
  - `content` (текст)
  - `timestamp` (время отправки)
  - `isDelivered` (флаг доставки)

- **Контакт** (`Contact`)
  - `userId` (владелец)
  - `contactId` (другой пользователь)
  - `lastMessageId` (последнее сообщение)

- **Соединение** (`Connection`)
  - `id`
  - `userId`
  - `peerAddress` (IP-адрес)
  - `port`
  - `startTime`
  - `endTime`

## Связи

- Пользователь **имеет** много Контактов (1 : N)
- Пользователь **отправляет** много Сообщений (1 : N)
- Пользователь **получает** много Сообщений (1 : N)
- Соединение **связывает** двух Пользователей (M : N)

## ER-диаграмма (Mermaid)

```mermaid
erDiagram
    User {
        int id PK
        string username
        string password
        string status
    }
    Message {
        int id PK
        int senderId FK
        int receiverId FK
        string content
        datetime timestamp
        bool isDelivered
    }
    Contact {
        int userId FK
        int contactId FK
        int lastMessageId FK
    }
    Connection {
        int id PK
        int userId FK
        string peerAddress
        int port
        datetime startTime
        datetime endTime
    }
    User ||--o{ Message : "отправляет"
    User ||--o{ Message : "получает"
    User ||--o{ Contact : "имеет"
    User ||--o{ Connection : "участвует"
