# Техническое задание (SRS)

## 1. Общие сведения

**Полное наименование системы:** Мобильный мессенджер с P2P-соединением.

**Сокращённое наименование:** P2P Messenger.

**Разработчик:** Студент группы [твоя группа], [твоё имя].

**Назначение:** Автоматизация процессов коммуникации между пользователями мобильных устройств без использования центрального сервера передачи сообщений.

**Основание для разработки:** Необходимость создания мобильного приложения для обмена текстовыми сообщениями в реальном времени с использованием прямого сетевого соединения между устройствами.

## 2. Назначение и цели создания системы

Целями разработки являются:

- создание мобильного приложения для обмена сообщениями между пользователями;
- организация прямого P2P-соединения между устройствами;
- обеспечение передачи сообщений в реальном времени;
- реализация хранения истории переписки;
- обеспечение удобного пользовательского интерфейса.

## 3. Характеристика объекта автоматизации

Объектом автоматизации является процесс обмена сообщениями между пользователями мобильного приложения.

Система должна обеспечивать:

- регистрацию пользователей;
- установление соединения между устройствами;
- передачу текстовых сообщений;
- хранение истории переписки;
- уведомление о новых сообщениях;
- отображение статуса соединения.

## 4. Требования к системе

Система должна обеспечивать:

- обмен сообщениями в режиме реального времени;
- прямое соединение между пользователями;
- локальное хранение переписки;
- отображение списка контактов;
- уведомления о новых сообщениях;
- обработку ошибок подключения.

## 5. Состав и содержание работ по созданию системы

Работы по созданию системы включают:

1. Анализ предметной области.
2. Разработку технического задания.
3. Проектирование архитектуры приложения.
4. Разработку пользовательского интерфейса.
5. Реализацию сетевого взаимодействия.
6. Тестирование функциональности приложения.
7. Подготовку документации.

## 6. Порядок контроля и приемки системы

Контроль и приемка системы выполняются путем проверки соответствия реализованных функций требованиям технического задания. Оцениваются:

- корректность подключения пользователей;
- корректность отправки и получения сообщений;
- стабильность работы приложения;
- сохранение истории сообщений;
- удобство интерфейса.

## 7. Требования к подготовке системы к вводу в эксплуатацию

Подготовка включает:

- установку приложения на мобильные устройства;
- настройку сетевого подключения (Wi-Fi или мобильные данные);
- проверку соединения между пользователями.

## 8. Требования к документированию

Система должна сопровождаться следующей документацией:

- руководство пользователя;
- описание структуры приложения;
- инструкция по установке и настройке.

## 9. Источники разработки

Источниками разработки являются:

- анализ существующих мобильных мессенджеров;
- технические возможности платформы Android;
- сетевые технологии передачи данных (TCP/IP, сокеты).
