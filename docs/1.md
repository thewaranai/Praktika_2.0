# Анализ предметной области

## Описание предметной области

**Мобильный мессенджер** — это программное средство, предназначенное для обмена сообщениями между пользователями в режиме реального времени с использованием мобильных устройств.

Современные мобильные мессенджеры обеспечивают передачу текстовых сообщений, мультимедийных файлов, уведомлений и поддерживают постоянную связь между пользователями. Наиболее распространённые решения основаны на клиент-серверной архитектуре, при которой передача сообщений осуществляется через центральный сервер, обеспечивающий маршрутизацию, хранение и синхронизацию данных между устройствами пользователей.

Альтернативным подходом является использование **peer-to-peer соединения**, при котором устройства пользователей взаимодействуют напрямую без промежуточного сервера передачи сообщений. Такая архитектура позволяет снизить нагрузку на серверную инфраструктуру, уменьшить задержки передачи сообщений и повысить автономность системы.

## Объект автоматизации

Объектом автоматизации является процесс обмена сообщениями между пользователями мобильного приложения.

Разрабатываемое приложение представляет собой мобильный мессенджер для платформы Android, использующий **P2P-модель обмена сообщениями**, позволяющую устанавливать соединение между пользователями напрямую через сетевые протоколы.

## Основные сущности предметной области

| Сущность | Описание |
|----------|----------|
| **Пользователь** | Лицо, использующее приложение для обмена сообщениями. Имеет уникальное имя, пароль, статус (онлайн/офлайн). |
| **Сетевое соединение** | Прямое P2P-подключение между двумя устройствами, характеризуется IP-адресом, портом, временем установки и завершения. |
| **Сообщение** | Единица обмена данными, содержит отправителя, получателя, текст, временную метку и статус доставки. |
| **История переписки** | Локально сохранённая коллекция сообщений между двумя пользователями. |

## Пользователи системы

Система предназначена для владельцев мобильных устройств под управлением **Android**. Пользователи осуществляют:

- регистрацию в приложении;
- подключение к другим пользователям;
- отправку и получение сообщений;
- просмотр истории переписки;
- управление контактами.

## Актуальность разработки

Разработка такого приложения актуальна в связи с необходимостью создания децентрализованных средств коммуникации, не требующих постоянного взаимодействия с серверной инфраструктурой. Это повышает приватность и надёжность обмена информацией.

## Аналоги и их недостатки

| Аналог | Недостатки |
|--------|------------|
| **WhatsApp** | Централизованный сервер, зависимость от интернета, сбор метаданных. |
| **Telegram** | Требует подключения к серверу, нет полной децентрализации. |

Разрабатываемое приложение лишено этих недостатков за счёт прямого P2P-соединения.
