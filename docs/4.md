# Описание прецедентов (Use Cases)

## Акторы
1. **Пользователь** – владелец мобильного устройства, использующий приложение.
2. **Система** – приложение-мессенджер, обеспечивающее функциональность.

## Прецеденты

### UC-1: Регистрация нового пользователя
**Актор:** Пользователь  
**Описание:** Пользователь вводит уникальное имя и пароль для создания учётной записи.  
**Предусловия:** Приложение запущено, пользователь не авторизован.  
**Постусловия:** Учётная запись сохранена локально, пользователь переходит в список контактов.  
**Альтернативный поток:** Имя уже занято – отображается сообщение об ошибке.

### UC-2: Вход в систему
**Актор:** Пользователь  
**Описание:** Пользователь вводит имя и пароль для входа.  
**Предусловия:** Приложение запущено, учётная запись существует.  
**Постусловия:** Пользователь авторизован, отображается список контактов.  
**Альтернативный поток:** Неверные данные – сообщение об ошибке.

### UC-3: Поиск доступных пользователей
**Актор:** Пользователь  
**Описание:** Пользователь инициирует поиск других устройств в локальной сети, готовых к соединению.  
**Предусловия:** Пользователь авторизован.  
**Постусловия:** Список обнаруженных пользователей отображается на экране.

### UC-4: Подключение к пользователю
**Актор:** Пользователь  
**Описание:** Пользователь выбирает из списка контакт и инициирует P2P-соединение.  
**Предусловия:** Список доступных пользователей получен.  
**Постусловия:** Устанавливается прямое сетевое соединение, статус меняется на «Подключён».  
**Альтернативный поток:** Соединение не установлено – отображается ошибка.

### UC-5: Отправка текстового сообщения
**Актор:** Пользователь  
**Описание:** Пользователь вводит текст и нажимает «Отправить». Сообщение передаётся через P2P-соединение.  
**Предусловия:** Активное соединение с собеседником.  
**Постусловия:** Сообщение отображается в истории переписки, доставляется собеседнику.  
**Альтернативный поток:** Соединение разорвано – сообщение не отправляется, выводится уведомление.

### UC-6: Получение сообщения (в реальном времени)
**Актор:** Система  
**Описание:** Приложение прослушивает входящие данные и при получении нового сообщения отображает его в интерфейсе.  
**Предусловия:** Соединение активно.  
**Постусловия:** Сообщение добавлено в историю, пользователь получает уведомление.

### UC-7: Просмотр истории переписки
**Актор:** Пользователь  
**Описание:** Пользователь открывает диалог с контактом и видит все ранее отправленные/полученные сообщения.  
**Предусловия:** Пользователь авторизован, имеются сохранённые сообщения.  
**Постусловия:** История отображается в хронологическом порядке.

### UC-8: Завершение соединения
**Актор:** Пользователь  
**Описание:** Пользователь нажимает кнопку «Отключиться» или закрывает диалог.  
**Предусловия:** Соединение активно.  
**Постусловия:** Соединение разрывается, статус меняется на «Отключён».
